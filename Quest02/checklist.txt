CSS를 HTML에 적용하는 세 가지 방법은 무엇일까요?
- 외부스타일시트
- 내부스타일시트
- 인라인시트

  세 가지 방법 각각의 장단점은 무엇일까요?
  - 한 css를 여러 문서에 적용할 수 있다
  - css파일을 수정할 수 없는 경우에 쓸 수 있지만 비효율적이다
  - 제한적인 상황이나 전자메일에서 쓸 수 있지만 비효율적이다

CSS 규칙의 우선순위는 어떻게 결정될까요?
- 우선순위가 동일한 경우 아래에 있는 것을 적용
- 더 구체적인 선택자가 우선순위가 더 높음(순서무시)
- id > class > tag

CSS의 박스모델은 무엇일까요? 박스가 화면에서 차지하는 크기는 어떻게 결정될까요?
- 블록 속성을 가진 요소가 한 줄을 차지함 마진-테두리-패딩-컨텐츠 = 화면

float 속성은 왜 좋지 않을까요?
- 수평정렬 할 때 플렉스, 그리드로 대체 가능하고 조정하기 어렵다

Flexbox(Flexible box)와 CSS Grid의 차이와 장단점은 무엇일까요?
- 플렉스 
- 랩을 이용해 화면 크기에 맞춰 열을 정리할 수 있다 중심을 맞추기 쉽다 플렉스 안에 플렉스 할 수 있다
- 여러줄로 나뉘면 일렬정렬하기 어렵다

- 그리드 
- 화면 크기가 달라져도 열 갯수를 지킨다 행 열의 크기와 비율 간격을 조절하기 쉽다
- 플렉스에 비해 덜 유동적이다(행/열 지킴)

CSS의 비슷한 요소들을 어떤 식으로 정리할 수 있을까요?
- 꾸밈요소
색/선/폰트/배경/이미지
- 디스플레이요소
블록/인라인/그리드/플렉스
- 정렬 요소
플로트/포지션
- 여백요소
마진/패딩/박스스타일
